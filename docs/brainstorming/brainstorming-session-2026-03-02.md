# Brainstorming Session Results

**Facilitator:** Michael
**Date:** 2026-03-02

## Session Overview

**Topic:** Music domain expert agent — deep music theory and instrumental knowledge spanning all eras
**Goals:** Define an agent with authoritative knowledge covering the Common Practice Period (Baroque, Classical, Romantic) through contemporary popular music (rock, jazz, electronic, hip-hop, etc.), with practical understanding of playing instruments — not just academic theory

### Context Guidance

_Focused on the Four Pillars of agent design: Identity (WHO), Voice (HOW), Purpose (WHAT), and Architecture (TYPE). Goal is to create an agent so vivid and useful that users seek it out by name._

### Session Setup

_Michael is building a music domain expert agent for music-related projects. The agent needs to bridge centuries of musical tradition — from counterpoint and figured bass to chord progressions in modern pop and jazz harmony. It must also understand the practical side: how instruments are played, their idiomatic qualities, and how to write effectively for them._

## Technique Selection

**Approach:** AI-Recommended Techniques
**Analysis Context:** Music domain expert agent with focus on authoritative era-spanning knowledge and practical instrumental expertise

**Recommended Techniques:**

- **Persona Journey:** Embody different musical archetypes (Baroque counterpoint master, jazz improviser, modern producer) to discover the agent's authentic voice across eras
- **Cross-Pollination:** Transfer concepts from real musical practice into agent capabilities — how musicians actually think, teach, and perform informs the agent's command set
- **Morphological Analysis:** Systematically map parameter dimensions (eras, instrument families, theory domains, interaction modes) to ensure comprehensive coverage and find optimal combinations

**AI Rationale:** The sequence moves from visceral identity exploration (who is this agent?) through practical capability generation (what does it do?) to systematic architecture mapping (how is it structured?) — mirroring how a musician develops from finding their voice, to building their repertoire, to mastering their craft.

## Technique Execution Results

**Persona Journey:**

- **Interactive Focus:** Explored Baroque Kapellmeister and jazz pianist archetypes to discover how the agent navigates different theoretical traditions
- **Key Breakthroughs:**
  - The agent is an anti-dogmatist — leads with acoustic/practical reasoning, not rules
  - Theory is multiple overlapping frameworks; the agent signals which one it's operating in
  - Parallel fifths are a contrapuntal concern, not a universal law — the agent knows this distinction
- **User Creative Strengths:** Deep historical precision, corrects oversimplifications, values "why it sounds that way" over "what the rule says"

**Cross-Pollination:**

- **Building on Previous:** Translated identity insights into concrete capabilities
- **Key Breakthroughs:**
  - The agent is a long-lived cross-project companion, not scoped to one project
  - Its user is a developer, not a musician — it translates domain knowledge into developer-actionable terms
  - Proactive assumption auditing is a core capability (the 12-TET lesson)
  - Domain concept mapping at project start prevents costly refactors
  - The agent watches during planning, steps back during implementation, jumps in during problem-solving

**Morphological Analysis:**

- **Building on Previous:** Systematically mapped all parameter dimensions
- **Key Findings:**
  - Knowledge: Essential — theory, tuning/intonation, instruments, notation. Moderate — performance practice. Low — history, acoustics
  - Functions: Core — answer questions, audit code/specs, validate correctness, flag assumptions. High — concept maps, content generation
  - Interaction: Active during planning & problem-solving. Off during implementation. Proactive in party mode
  - Architecture: No sidecar. Self-contained. Reads project context from documentation

**Overall Creative Journey:** Session moved efficiently from identity exploration through capability generation to systematic architecture decisions. Michael's deep domain knowledge shaped the agent into something precise and practical rather than broad and vague.
