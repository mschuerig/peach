## Commit message

Fix CSV export timestamp truncation breaking merge duplicate detection

Export used date.formatted(.iso8601) which drops sub-second precision.
Imported timestamps then differed from originals, defeating dedup on
first re-import. Switch to ISO8601FormatStyle(includingFractionalSeconds: true).

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
