**Audit Framework: Adam's #audit-assumptions process**


Starting with `Interval.swift`, I'm examining the interval classification system. The enum correctly maps the 13 cases from P1 through P8 to represent all simple intervals within the 12-TET chromatic scale, which aligns with standard Western music theory. However, there's a significant limitation: the implementation doesn't account for compound intervals beyond the octave, yet the app's MIDI range spans over 10 octaves, creating a gap between what the code supports and what the system actually needs to handle.

There's also a deeper assumption baked into the design—the enum treats intervals as purely semitone counts, which works perfectly in 12-TET but obscures how intervals are actually defined in other tuning systems. In just intonation or Pythagorean tuning, intervals are frequency ratios like 3:2 or 5:4, not semitone distances. So while the enum functions as a chromatic classifier, the naming conventions (like "Major Third" = 4 semitones) are really 12-TET approximations that don't capture the true acoustic definition of these intervals in alternative tuning systems.

The `between()` method using `abs()` is correct but intentionally discards direction—it returns unsigned distance, which is musically appropriate since "interval" on its own means distance, not direction. That distinction is properly handled by the layered design where `DirectedInterval` captures both. When reference equals target, `abs()` correctly returns 0 and maps to `Interval.prime`.

The tritone abbreviation as "d5" is raising some flags though—I need to think through the standard music theory naming here. iguous in classification:
     - **Diminished fifth (d5)**: C → G♭ — a fifth made smaller
     - **Augmented fourth (A4)**: C → F# — a fourth made larger
   - These are enharmonically equivalent in 12-TET but conceptually different. The choice of "d5" privileges one interpretation.
   - For an ear training app that doesn't deal with harmonic context or key signatures, this is a pragmatic choice. The localized name "Tritone" is the neutral term.
   - **Recommendation**: Consider "TT" (tritone) as a more neutral abbreviation, or keep "d5" with awareness that it's a 12-TET simplification.

Now I'm looking at the `DirectedInterval` implementation, which handles intervals with directional information.
