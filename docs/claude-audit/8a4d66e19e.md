## Commit message

Implement story 38.3: ProgressChartView, Profile Screen redesign, and review fixes

Add ProgressChartView with Apple Charts (EWMA line, stddev band, optimal baseline).
Redesign ProfileScreen to card-based layout using ProgressTimeline as sole data source.
Delete old profile views (ThresholdTimelineView, SummaryStatisticsView, MatchingStatisticsView)
and old core types (ThresholdTimeline, TrendAnalyzer). Decouple TrainingDataTransferService
from profile/progressTimeline via onDataChanged callback. Move mode-specific metric extraction
from ProgressTimeline into TrainingMode so adding modes doesn't require ProgressTimeline changes.
Use Cents and Duration types in TrainingModeConfig, add German translations for mode names.
