# Tech-Spec: Settings Sound Source Preview Button

**Created:** 2026-03-06

## Overview

### Problem Statement
There is no way to hear what a sound source sounds like before starting a training session. Users must leave Settings, start training, and listen — then go back to Settings if they want a different sound.

### Solution
Add a `speaker.wave.2` icon button to the right of the Sound picker in the Settings screen. Tapping it plays a 2-second preview of the selected sound at the current concert pitch (A4). Tapping again while playing stops the preview. The preview action is injected via `@Environment` as a closure from the composition root, keeping `NotePlayer` out of the view layer.

### Scope
**In Scope:**
- Preview button with `speaker.wave.2` SF Symbol icon next to the Sound picker
- Plays selected sound source at concert pitch (A4) for 2 seconds
- Tap-to-stop if already playing
- Preview closure injected via `@Environment` from composition root
- Constant for preview duration

**Out of Scope:**
- Visual feedback during playback (e.g., animated speaker icon)
- Preview for other settings (duration, tuning system)
- Playing multiple notes or a sequence

## Implementation Plan

### Tasks

1. **Add `previewDuration` constant** — `Peach/Core/Training/TrainingConstants.swift`
   - Add `static let previewDuration: TimeInterval = 2.0`

2. **Add environment keys** — `Peach/App/EnvironmentKeys.swift`
   - `@Entry var soundPreviewPlay: (() async -> Void)? = nil`
   - `@Entry var soundPreviewStop: (() async -> Void)? = nil`

3. **Wire closures in composition root** — `Peach/App/PeachApp.swift`
   - Store `notePlayer` as `@State` property (currently local `let` in `init()`)
   - Create play closure: computes `TuningSystem.equalTemperament.frequency(for: MIDINote(69), referencePitch: userSettings.referencePitch)`, calls `notePlayer.play(frequency:duration:velocity:amplitudeDB:)` with `TrainingConstants.previewDuration`, `TrainingConstants.velocity`, `AmplitudeDB(0)`
   - Create stop closure: calls `notePlayer.stopAll()`
   - Add `.environment(\.soundPreviewPlay, ...)` and `.environment(\.soundPreviewStop, ...)` to `ContentView()` chain

4. **Add preview button to Settings** — `Peach/Settings/SettingsScreen.swift`
   - Add `@Environment(\.soundPreviewPlay)` and `@Environment(\.soundPreviewStop)`
   - Add `@State private var previewTask: Task<Void, Never>?`
   - Wrap Sound `Picker` in `HStack` with preview `Button` using `speaker.wave.2` / `stop.fill` icon
   - Add `togglePreview()` method (start task or cancel + stop)
   - Stop preview on `.onDisappear` and `.onChange(of: soundSource)`

### Acceptance Criteria

1. **AC 1:** Given the Settings screen is showing, when the user looks at the Sound section, then a `speaker.wave.2` icon button is visible to the right of the Sound picker
2. **AC 2:** Given no preview is playing, when the user taps the preview button, then a 2-second note plays using the currently selected sound source at the current concert pitch
3. **AC 3:** Given a preview is playing, when the user taps the preview button again, then the playing sound stops immediately
4. **AC 4:** Given a preview is playing, when the user changes the sound source in the picker, then the playing preview stops
5. **AC 5:** Given a preview is playing, when the user navigates away from Settings, then the playing preview stops
6. **AC 6:** Given the preview finishes its 2-second duration naturally, then the button returns to its default `speaker.wave.2` icon

### Testing Strategy

**Unit tests:** `TrainingConstants.previewDuration` equals `2.0`
**Integration tests (manual):** Tap preview → hear sound for 2s; switch source → hear new sound; tap during playback → stops; navigate away → stops; change concert pitch → pitch changes

### Notes
- `NotePlayer` is shared with training sessions, but Settings is only accessible when no session is active, so no conflict
- `SoundFontNotePlayer.ensurePresetLoaded()` auto-loads the correct preset on each `play()` — changing sound source in the picker then tapping preview just works
- Preview uses `equalTemperament` for frequency — for A4, tuning system makes no difference (it's the reference pitch in all systems)
