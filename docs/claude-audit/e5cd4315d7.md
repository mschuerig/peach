### Key Changes

1. **`UserSettings` protocol** gained `intervals: Set<Interval>` and `tuningSystem: TuningSystem` properties. `AppUserSettings` returns hardcoded `[.perfectFifth]` / `.equalTemperament`. `MockUserSettings` defaults to `[.prime]` / `.equalTemperament` ensuring all existing tests work unchanged.

3. **`startTraining()` renamed to `start()`** across production code and ~80 test call sites. `start()` reads interval context from `userSettings` with a non-empty precondition.

4. **Observable `currentInterval` and computed `isIntervalMode`** properties added to `ComparisonSession`. Cleared on `stop()`.

5. **All 3 hardcoded `.equalTemperament`** references in `ComparisonSession` replaced with session-level `sessionTuningSystem`.
