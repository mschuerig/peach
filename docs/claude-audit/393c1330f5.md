## Commit message

Extract TrainingDataTransferService from SettingsScreen

Move import/export orchestration (file reading, CSV parsing, error
formatting, import dispatch, profile/trend/timeline rebuild, export
caching) into a dedicated service in Core/Data/. SettingsScreen now
delegates all data transfer operations to the service via @Environment.

Remove unused environment closures (trainingDataExportAction,
csvExportDocumentAction, trainingDataImportAction) from EnvironmentKeys.

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
