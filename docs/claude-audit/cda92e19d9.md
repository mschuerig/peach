The real dependency is `TrainingDataStore`, which is already a concrete class without a protocol. For testability, I could either pass records as parameters or inject the store as a dependency, and since we already have in-memory SwiftData for tests, taking the store dependency seems like the more straightforward approach.

Building out the exporter as an enum with a static method that fetches all comparisons and pitch matchings from the store, merges them by timestamp, and formats each record as CSV...

The exporter needs to use MainActor isolation since it calls MainActor methods on TrainingDataStore, so I'll structure it as a simple enum with a static export method that takes the store as a parameter.
