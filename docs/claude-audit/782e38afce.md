### H1: `tuningSystem` not added to `NextComparisonStrategy` protocol — Task 2 marked [x] but not fully done

**File:** `Peach/Core/Algorithm/NextComparisonStrategy.swift:38-43`
**AC #5** states the protocol gains "`interval: Interval` and `tuningSystem: TuningSystem` parameters (no defaults)". Only `interval` was added.

**Task 2** claims `[x] Update protocol signature: nextComparison(profile:settings:lastComparison:interval:tuningSystem:)` — this is false. The completion notes also state "Updated `NextComparisonStrategy` protocol with `interval` and `tuningSystem` parameters" which is inaccurate.

**Architectural assessment:** The omission is actually the **correct architectural decision**. The strategy operates entirely in the logical world (MIDINote, DetunedMIDINote, Cents). `TuningSystem` is a physical-world bridge parameter that the strategy never uses. Passing it would violate the two-world architecture principle documented in `project-context.md`. However, **the story documentation must be corrected** to reflect this intentional deviation from the AC.

**Action needed:** Update story Task 2 subtasks and completion notes to accurately describe what was implemented. Optionally update AC #5 to remove the `tuningSystem` requirement from the strategy protocol.

**File:** `PeachTests/Comparison/ComparisonSessionTests.swift:244-256`

### M3: `PreviewComparisonStrategy` ignores `interval` parameter — preview inconsistency

**File:** `Peach/App/EnvironmentKeys.swift:62-71`
