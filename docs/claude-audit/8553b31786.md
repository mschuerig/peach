If the Arguments tab is gone in Xcode 26, environment variables for tests are likely managed through **Test Plans** now. Here's how:

1. **Product > Test Plan > Edit Test Plan** (or click the test plan name in the Test navigator)
2. Select the **Configurations** tab
3. Add `RUN_STRESS_TESTS` = `1` under **Environment Variables**

If you don't have a test plan yet, create one via **File > New > File > Test Plan**, then associate it with the Peach scheme.

Alternatively, the command line always works and doesn't need any Xcode UI:

RUN_STRESS_TESTS=1 xcodebuild test -scheme Peach -destination 'platform=iOS Simulator,name=iPhone 17'
