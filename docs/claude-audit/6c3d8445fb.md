### Summary

**Story:** 23-3-pitchmatchingsession-interval-parameterization
**Title:** PitchMatchingSession Start Rename, Interval Support, and Protocol Update

### Key Changes

1. **`startPitchMatching()` → `start()`** — Renamed and now reads `intervals` + `tuningSystem` from `userSettings` with non-empty precondition
2. **Interval-aware challenge generation** — `generateChallenge(settings:interval:)` constrains MIDI range by `127 - interval.semitones`, computes `targetNote` via `transposed(by:)`. Unison (`.prime`) is identical to prior behavior.
3. **Session tuning system** — All 3 hardcoded `.equalTemperament` replaced with `sessionTuningSystem`. `CompletedPitchMatching` carries the session's tuning system.
4. **Target note anchor** — `referenceFrequency` now stores the **target note** frequency (not reference note), so `userCentError` correctly measures deviation from the interval target. For unison, this is unchanged.
5. **Observable state** — `currentInterval` and `isIntervalMode` exposed for UI (Story 23.4)
6. **TrainingSession protocol** — `start()` added; both `ComparisonSession` and `PitchMatchingSession` satisfy it
7. **Test rename** — 46 occurrences of `startPitchMatching()` → `start()` across test files, plus 11 new interval/protocol tests
