<command-args>I want to introduce a variant to each of the existing training types. Currently, they are concerned with the pitch of one target note: either to decide whether a second note is higher/lower, or to match the target note. In both cases, the target note is played first. Here's the variation I want to introduce:

Comparison Training: Given a musical interval. note1 is played. note2 is played at the given interval _plus_ a signed cent difference (as before). The user's task is to judge whether note2 was higher or lower.

Pitch Matching Training: The reference note is played. The user has to adjust the second note to match a given interval.

There are some extensions I can think of that we will need:

* An Interval value object for representing intervals from Prime through Octave.
* Settings for choosing which intervals are to be used in training; separate for up and down direction. Here we will need to do some UI design for the view.
* Several datastructure will have to be extended.
* Several of the training classes/methods will need to take a parameter with a set of available intervals.
* On the start screen, these training variants should get two new buttons. The existing buttons should use the same underlying training sessions and screen, just fixed to a prime interval.

We will have to introduce Tuning Systems. For a start, we can use https://en.wikipedia.org/wiki/12_equal_temperament (in fact that is what we are implicitly using now), but we need to support more. At least Just Intonation.

I'd like to get a usable and testable implementation quickly, so we should start with a single, fixed interval: perfect fifth up.</command-args>
