2. **`precondition()` in FrequencyCalculation.swift:60** - MIDI note validation uses a precondition that gets stripped in optimized Release builds, and the error handling should use proper exception throwing instead of crashing. 3. **`preconditionFailure()` in InfoScreen.swift:12** - URL validation crashes if the constant is malformed, which is a crash path that shouldn't exist in production code.

4. **`nonisolated(unsafe)` in 3 environment keys** - The environment keys bypass actor isolation safety by telling the compiler to trust the isolation, which creates undefined behavior if accessed off the main actor, though this pattern is necessary due to SwiftUI's constraints.

5. **Untracked `Task` in HapticFeedbackManager.swift:41** - Fire-and-forget haptic tasks can accumulate if rapid incorrect answers trigger multiple impacts without tracking.

6. **Untracked `Task` in TrainingSession.stop():286** - The note player stop method runs in an untracked background task.

7. **Untracked `Task` in TrainingSession.handleAnswer():239** - Similar fire-and-forget pattern for early note stopping.

8. **MockHapticFeedbackManager in production code** - Mock implementations are shipping in the main target instead of being isolated to tests, and they're also referenced in preview code within the production view file.

9. **Preview mocks mixed into production views** - Mock objects for testing are defined directly in the production view file rather than separated into test utilities.

10. **SettingsScreen creating its own TrainingDataStore** - The settings view is instantiating a service directly, which contradicts the established pattern that all service creation should happen in PeachApp.swift.

3. **Three environment keys use `nonisolated(unsafe)` with `MainActor.assumeIsolated()` — a pattern that bypasses actor isolation.** Found in `TrainingScreen.swift:152`, `ProfileScreen.swift:149`, and `TrendAnalyzer.swift:93`.
